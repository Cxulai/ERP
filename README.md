# 📊 ERP 企业管理系统 - 使用手册

## 一、系统简介

一个轻量级的企业资源计划（ERP）管理系统，采用 Python Flask + SQLAlchemy 构建，支持 SQLite（本地）和 PostgreSQL（云端部署），适合小微企业日常经营管理使用。

### 核心功能模块

| 模块 | 功能描述 |
|------|---------|
| 📈 **仪表盘** | 关键经营指标一览：销售额、采购额、库存、客户/供应商统计、销售趋势图、低库存预警 |
| 📦 **商品管理** | 商品的增加、修改、删除、搜索，库存手动调整 |
| 🏗️ **库存流水** | 所有出入库记录追踪，可追溯到关联订单 |
| 👥 **客户管理** | 客户信息的增删改查 |
| 🏭 **供应商管理** | 供应商信息的增删改查 |
| 💰 **销售订单** | 销售订单全生命周期：创建→确认→发货→完成，支持自动扣减库存 |
| 🛒 **采购订单** | 采购订单全生命周期：创建→确认→收货→完成，收货自动增加库存 |
| 📋 **财务报表** | 月度收入/支出/利润汇总，应收应付账款，销售采购趋势图表 |

---

## 二、环境要求

- **Python** 3.8 或以上版本
- **操作系统**: Windows / macOS / Linux 均可

---

## 三、安装与启动

### 3.1 首次安装

打开终端（PowerShell / CMD），进入项目目录：

```bash
cd erp-system
```

安装依赖：

```bash
pip install -r requirements.txt
```

### 3.2 启动系统

```bash
python app.py
```

启动成功后，终端会显示：

```
 * Running on http://0.0.0.0:5000
```

### 3.3 打开浏览器

在浏览器地址栏输入：**http://localhost:5000**

> 💡 首次启动会自动创建数据库并填充示例数据（8 个商品、4 个客户、3 个供应商、5 个销售订单），方便快速体验。

## 四、功能使用说明

### 4.1 仪表盘

进入系统后默认显示仪表盘页面，包含：

- **4 个 KPI 卡片**：本月销售额、本月采购额、商品总数、客户/供应商数量
- **月度销售趋势图**：柱状图展示最近月份销售情况
- **低库存预警表**：库存量低于最低库存设置的商品列表
- **最近销售订单**：最新的 5 笔销售订单

### 4.2 商品管理

点击左侧菜单「📦 商品管理」进入。

**新增商品**：
1. 点击右上角「+ 新增商品」按钮
2. 填写商品编码、名称、分类、单位、销售价、采购价、初始库存、最低库存预警值
3. 点击「保存」

**编辑商品**：在商品列表中点击「编辑」按钮，修改后保存。

**库存调整**：
1. 点击「库存」按钮
2. 选择调整类型（入库 / 出库 / 手动调整）
3. 输入变动数量（出库填写负数）
4. 填写备注原因
5. 点击「确认调整」

**删除商品**：点击「删除」按钮（已被订单引用的商品无法删除）。

**搜索**：使用表格上方的搜索框按名称或编码搜索。

### 4.3 库存流水

点击左侧菜单「🏗️ 库存流水」查看所有库存变动记录，包括：

- 变动时间和商品信息
- 变动类型（初始库存、入库、出库、销售出库、采购入库、取消退回等）
- 变动前后的库存数量对比
- 关联的订单号

### 4.4 客户管理

点击左侧菜单「👥 客户管理」进入。

- 支持新增、编辑、删除客户
- 客户信息包括：名称、联系人、电话、邮箱、地址
- 已被销售订单引用的客户无法删除
- 支持搜索过滤

### 4.5 供应商管理

点击左侧菜单「🏭 供应商管理」进入。

- 操作方式与客户管理相同
- 记录供应商的名称、联系人、电话、邮箱、地址

### 4.6 销售订单

点击左侧菜单「💰 销售订单」进入。

**订单状态流转**：

```
草稿 ──确认──▶ 已确认 ──发货──▶ 已发货 ──完成──▶ 已完成
  │                                    │
  └────────── 取消 ────────────────────┘
  (任意非完成状态均可取消)
```

**创建销售订单**：
1. 点击「+ 新建销售订单」
2. 选择客户和订单日期
3. 点击「+ 添加商品」添加订单行
4. 选择商品后单价自动填入（销售价），可手动修改
5. 系统自动计算合计金额
6. 点击「创建订单」

**订单操作**：
- **确认**：将草稿订单确认，此时会自动扣减库存（如库存不足会提示错误）
- **发货**：标记订单为已发货状态
- **完成**：完成订单，自动将已付金额设为总金额
- **取消**：取消订单，如果已确认则会自动退回库存
- **编辑**：仅草稿状态可编辑
- **删除**：仅草稿和已取消状态可删除

**筛选**：可按订单状态（草稿/已确认/已发货/已完成/已取消）筛选。

### 4.7 采购订单

点击左侧菜单「🛒 采购订单」进入。

**订单状态流转**：

```
草稿 ──确认──▶ 已确认 ──收货──▶ 已收货 ──完成──▶ 已完成
  │                                    │
  └────────── 取消 ────────────────────┘
  (仅草稿和已确认状态可取消)
```

**创建采购订单**：与销售订单操作类似，但选择的是供应商，商品单价自动填入采购价。

**订单操作**：
- **确认**：确认订单（不改变库存）
- **收货**：收货后**自动增加库存**
- **完成**：完成订单，自动将已付金额设为总金额
- **取消**：取消订单（如已收货则不可取消）

### 4.8 财务报表

点击左侧菜单「📋 财务报表」进入。

**概览指标**：
- 本月销售收入、采购成本、毛利
- 应收账款、应付账款
- 待处理销售额

**趋势图表**：
- 销售收入趋势（按月，区分已完成和总计）
- 采购支出趋势（按月，区分已完成和总计）

## 五、业务逻辑说明

### 库存联动规则

| 操作 | 库存变化 |
|------|---------|
| 销售订单确认 | 扣减库存（按订单明细） |
| 销售订单取消（已确认/已发货） | 退回库存 |
| 采购订单收货 | 增加库存（按订单明细） |
| 手动库存调整 | 增加/减少库存 |

### 订单金额说明

- **total_amount**：订单总金额（自动计算 = Σ(数量 × 单价)）
- **paid_amount**：已付金额
- 订单完成时自动将 paid_amount 设为 total_amount

## 七、常见问题

**Q: 如何重置所有数据？**
A: 关闭程序，删除项目目录下的 `erp.db` 文件，重新运行 `python app.py`。

**Q: 端口被占用怎么办？**
A: 编辑 `app.py` 文件，修改 `port=5000` 为其他端口号（如 `port=8080`）。

**Q: 能否在局域网内其他电脑访问？**
A: 可以。启动后，局域网内其他电脑通过 `http://你的IP:5000` 访问（如 `http://192.168.1.100:5000`）。

**Q: 删除订单时提示"只能删除草稿或已取消的订单"？**
A: 这是为了防止误删。请先将订单取消，再删除。

## 八、技术架构

```
前端: HTML5 + CSS3 + Vanilla JavaScript + Chart.js
后端: Python 3 + Flask
数据库: SQLite3 (本地) / PostgreSQL (Vercel 云端)
架构: SPA (Hash 路由) + RESTful API + SQLAlchemy ORM
```
